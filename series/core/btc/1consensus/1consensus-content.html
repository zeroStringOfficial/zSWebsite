Bitcoin is a new form of money; one that no one controls and that exists solely in cyberspace. These unique attributes foster much intrigue and speculation about the digital asset, raising many questions: if no one controls Bitcoin, how does it work? Who determines what transactions are genuine? The question this article will aim to cover is “Who determines what transactions are genuine?” By exploring this question, we'll illuminate a key component of Bitcoin's architecture, contributing to our understanding of the larger, encompassing question: “How does Bitcoin work?"

This article begins with a technical definition of Bitcoin, which we'll dissect and explain. The aim of the article is that by the end, you'll understand this technical definition and gain insight into Bitcoin's key inner workings. The definition is as follows: at its core Bitcoin is a shared ledger, where transactions recorded to the ledger are validated through a ‘consensus mechanism’ called Proof-of-Work (PoW). To fully grasp this definition, let's break down some key terms.

Checkpoint 1:

  • Ledger: A record of transactions (in Bitcoins case, Alice sends Bob 0.5 BTC).
  • Shared/distributed ledger: A shared ledger also known as a ‘distributed ledger' is a system where many independent computers (called 'nodes') across the network maintain identical copies of the ledger, enhancing security and eliminating the need for a central authority.
  • Blockchain: Satoshi's adaptation of a shared ledger, where transactions are recorded in bundles called blocks. These blocks of transactions are validated, rather than validating and appending each transaction individually.
  • Consensus mechanism: a mechanism or piece of code that allows us to agree on the legitimacy of transactions. 

Now that we have clarified these fundamental concepts, let's dive deeper into the heart of Bitcoin's innovation: its consensus mechanism.

Why do we need ‘Proof-of-Work’ and how does it actually work? 

In a distributed system such as Bitcoin, participants need to vote on transactions they believe are valid or invalid. But the challenge is how do we create a voting system where it is very difficult to spin up fake votes, while not destroying the ability to vote legitimately? There are a few answers to this question, and Satoshi Nakamoto’s answer was Proof-of-Work. A system that uses participants' energy and hardware to back their voting power, making it difficult and costly to manipulate the network. This is essential to the integrity of the Bitcoin network, as the ability to create fake votes is the ability to outvote others, and the ease of outvoting others threatens the security and reliability of all transactions.

With the rationale behind Proof-of-Work established, let's delve into the mechanics of how this approach actually works. The phrase ‘Miners compete to solve complex mathematical puzzles’ is an exhausted oversimplification that leaves users no more educated than before they read it. Instead, we will start from the beginning with how mining works succinctly, then move on to how mining allows consensus. To start, Bitcoin’s security relies on a hashing function called SHA-256. However, this raises important questions: What is a function? What is a hashing function?

A function in computer science is where an input is taken in, a process is applied to the input, which then produces an output, like so:

In the example below, a hashing function such as SHA-256 takes any input (orange), whether it's a simple word or a complex file, and creates a unique same-length output, called a hash (green), as shown below:

A key feature of SHA-256 is that any change in the input, no matter how small, results in a radically different output. In the figure above, the inputs ‘hello’ and ‘hello1’ produce completely different hashes despite being similar inputs. This means you can take any base piece of information you want, i.e., ‘hello’, and manipulate the output by adding numbers to the end. The number set to manipulate is called a ‘nonce’. When we increase the nonce from hello1 to hello2, this is called incrementing.

Incrementing the nonce to achieve a specific output is crucial in Bitcoin transaction verification because this process aims to meet the 'mining difficulty' set by the Bitcoin network. By adding a number, or 'nonce,' miners manipulate the outputted hash to satisfy the required difficulty level. While mining difficulty is a long-winded topic that will be discussed in a separate article, it can be summarised as follows: the network sets a difficulty by saying “We want five zeros at the beginning of the hash produced”. This requirement forces miners to make numerous guesses to find a valid hash that meets the criteria. The more zeros required at the beginning of the hash, the more guesses are needed, as finding a valid hash becomes increasingly unlikely, thus increasing the ‘mining difficulty’. In the example above, ‘hello28’ results in a hash starting with 1 zero (gold). If the difficulty was set to 1 zero, then this would be a valid hash, as the required information ‘hello’ combined with the correct number ‘28’ yielded a result that satisfies the difficulty requirement.

Having a target of 1 zero, and incrementing the nonce until a hash meets the target: this is called mining. It isn’t Bitcoin mining just yet, however the logic and workings of how mining is done like this step by step.

Checkpoint 2:

  • Function: where an input is taken in, a process is applied to the input, and an output is produced. A simple example of a function could be that the input is the number 2, the process is a 2x multiplier, so the output is 4. A more complex example of a function is a hashing function, shown in the next point.
  • Hashing function: a type of function that takes an input, processes it through a hashing algorithm and produces a fixed-size (eg. 64 characters long) output called a hash.
  • Nonce: a number that is hashed along with the block constituents to manipulate the resulting output. If the hash produced doesn’t meet the target, add 1 or ‘increment’ the nonce and try again.
  • Mining: hashing information along with a nonce to manipulate the output until the mining difficulty has been satisfied.
  • Mining difficulty: the number of zeros required at the beginning of a hash. The more zeros required, the less likely it is to meet the difficulty.

So why does forcing participants to mine or guess hashes help us agree on transactions?

Now that we know what mining is, where we combine information with a nonce to change the outputted hash. We need to take the next step into what exactly Bitcoin mining is; it is more complicated, but it is the same idea. Instead of the example above where the information ‘hello’ is combined with a nonce, we instead combine ‘block constituents’ with a nonce. These constituents are all separate pieces of information that are aggregated and united to wholly form a block. These constituents are combined and hashed together, miners then add a nonce to this combination and hash it again to meet the mining difficulty. If the difficulty is met, only then may it be proposed for the blockchain. Let's visualise these constituents and break them down:

This is how Bitcoin mining truly works.

How does it help us agree on transactions? Because this is how you vote within the Bitcoin network. You must mine blocks containing thousands of transactions to propose them to the network. As we said earlier, you need a mechanism in which you make fake voting difficult and costly. For someone to consistently outvote others on which transactions are to be included in Bitcoin’s blockchain/ledger, they would have to mine blocks faster than any other miner in the world consistently. The only way to do this would be to have 51% of Bitcoin's entire network power, which would require tens of billions of USD in hardware and energy.

Now that we have explored the heart of Bitcoin’s consensus mechanism, we can revisit our original description of the currency: at its core Bitcoin is a shared ledger, where transactions recorded to the ledger are validated through a ‘consensus mechanism’ called Proof-of-Work (PoW).
